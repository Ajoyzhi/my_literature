FDDA 算法

1、符号说明

  • ：过滤可疑请求的门限值，单位：请求数/s，超过为可疑请求；
  • 黑名单(BL)：，分别表示发送可疑请求的IP地址、可疑的请求、可疑请求的发送频率；
  • ：来自同一主机的不同请求之间的关联度，请求和可表示为，使用 关联规则算法 计算。如果是正常连接，一般会发送大量不同的连接，即会出现一个请求出现之后，又出现另一个连接的情况；而僵尸主机不会出现这种情况；
  • ：来自同一主机的请求频率之间的相似性[请求数量/时间间隔]。正常主机是有不同的请求需求的，所以相同时间内请求数量（这里不区分什么请求）一般不同；而僵尸主机发送的请求数量一般都是固定的，使用 DBSCAN算法聚类不同的请求频率；
  • ：记录所有发送给服务器的请求，组成形式（源IP地址；请求本身；请求频率）；
  • ：记录所有可疑的请求，是黑名单更新的标准；
  • ：抽样间隔，每过时间，就进行一次检测算法的进行，由函数F(t)决定；
  • ：与系统状态有关，如果系统正常，大一些，不用那么频繁地进行检测，如果系统异常，小一些，要频繁检测。

2、算法说明
（1）F(t)

  • 如果为空，说明当前系统中没有可疑的请求，为安全状态，抽样的时间间隔可以逐渐增加；
  • 如果为不空，说明系统中存在可疑的请求，所以抽样的时间间隔要调为最小，密切关注网络中请求的状态。

  • 将请求记录在中，然后通过 发送频率 进行筛选，满足条件的请求（+ 源IP）放入中。再通过和，从中筛选出正常的请求，并从中删除。最后根据更新黑名单。
