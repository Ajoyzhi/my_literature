Software Defined Networking（SDN）

各层易遭受的攻击：

  • 数据层：DoS、DDoS、数据修改、黑洞攻击、信道攻击；
  • 控制器：DoS、DDoS、黑洞攻击、生成假流规则、单点失效；
  • 应用层：要具体到应用。

Distributed Denial of Service（DDoS）Attack

其中提到一个信息安全报告，我觉得是最有用的内容了。
（1）DDoS攻击分类：

  • 基于数量：发送大量数据包，以占用资源，可以使用“每秒的字节数”检测；
  • 基于协议：通过网络协议的漏洞发动攻击，最终可能会导致底层的操作系统崩溃，可以使用“每秒的数据包数”检测；
  • 基于应用：通过应用层协议发生攻击，可以使用“每秒的请求数”检测。

（2）常见的DDoS攻击：

  • SYN flooding攻击；
  • UDP flooding攻击：针对victim的不同端口发送大量攻击数据包，发现空闲端口；
  • ICMP flooding攻击：发送大量ICMP echo数据包；
  • HTTP flooding攻击；
  • ping of death攻击；
  • smurf攻击：smurf为一种攻击病毒，攻击者将源IP设置为victim_IP，广播ICMP请求到网络中，当接收到该请求后，网络中的所有主机发送ICMP回复给victim_IP，使victim宕机；
  • slowloris攻击：以极低的速度向服务器发送HTTP请求，并一直恶意占据资源，导致服务器拒绝服务。

The Process of Detection

（1）流量生成：Scapy

  • UDP flooding流量：victim_IP固定，源端口和目的端口为1-80，产生速率为2000packets/min；
  • TCP flooding流量：victim_IP、victim_port和数据包数量固定；
  • 正常流量：100packets/s，大概接近正常流量的速率。

（2）流量收集：通过交换机收集“主机发送的数据包数量”、“主机使用的bytes数”、“发送和接收数据包的持续时间”。

总结

（1）自己产生的数据不能模拟真实环境下的流量特征，可能会比较简单，所以检测的精确度才可能达到97%之高。但是，该系统只要具有线上训练的能力，即使使用真实流量，也可以抽取真实流量的特征，对真实流量进行分类，所以该方法中使用自己产生的流量进行训练并测试没有问题，只是真实情况下，检测结果可能比现在差。
（2）该论文没有给我太大的启发，但是那个信息安全报告可以看一下，可能会存在一些有用的数据。
